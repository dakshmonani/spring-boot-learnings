Deep Dive Into Each File
🚫
AuthEntryPointJwt.java
implements AuthenticationEntryPoint
Error Handler
Think of this as the airport officer who stops you at the gate. When someone tries to access a protected endpoint without a valid token, Spring Security automatically calls this class. Its job is simple: send back a clean, structured JSON error instead of a raw ugly server error.
commence()
Called automatically by Spring Security when an unauthenticated user hits a protected route. Sets response status to 401.
🚫 "Officer's standard rejection"
// What it sends back to the client { "status": 401, "Error": "Unauthorized", "Message": "Full authentication is required to access this resource", "path": "/api/products" }
🏭
JwtUtils.java
@Component — Token Factory & Inspector
Core Logic
This is the passport office of your JWT system. It does three things: issue passports (create tokens), read passports (decode tokens), and verify passports (validate tokens). It reads your secret key and expiry time from application.properties via @Value.
key()
Converts your Base64-encoded secret string from application.properties into a real cryptographic signing key.
🔏 "The government's official seal"
generateTokenFromUserName()
Takes a logged-in user's details, embeds their username + timestamps, signs it with the secret key, and returns a compact JWT string.
🛂 "Issuing a new passport"
getJwtFromHeader()
Reads the Authorization header from the HTTP request and strips the "Bearer " prefix to extract the raw JWT token.
✋ "Asking to see the passport"
getUserNameFromJwtToken()
Decodes the JWT token and extracts the subject field, which is the username you embedded when generating the token.
📖 "Reading the name inside it"
validateJwtToken()
Tries to parse the JWT. Catches 4 specific exceptions: expired, malformed, unsupported, or empty — logs each and returns false. Returns true only if valid.
🔦 "Scanner checking if it's genuine"
// Token structure — what's inside your JWT // HEADER (algorithm used) { "alg": "HS256" } // PAYLOAD (your data — the "claims") { "sub": "john_doe", "iat": 1720000000, "exp": 1720086400 } // SIGNATURE (tamper-proof seal) HMACSHA256(base64(header) + "." + base64(payload), secretKey) // Final token sent to client looks like: eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJqb2huX2RvZSJ9.xyz123...
🚦
AuthTokenFilter.java
extends OncePerRequestFilter
Request Interceptor
This filter runs on every single incoming HTTP request — once and only once (that's what OncePerRequestFilter guarantees). Think of it as the security scanner at the airport boarding gate: every passenger (request) must pass through it before reaching their flight (controller).
doFilterInternal()
The main checkpoint logic. Runs on every request. If a valid JWT is found, it loads the user and sets them as authenticated in Spring's SecurityContext.
🔍 "The full boarding gate scan"
parseJwt()
A private helper method. Simply delegates to JwtUtils.getJwtFromHeader() to extract the token from the request. Keeps code clean and readable.
✋ "Ask the passenger for their passport"
// doFilterInternal — simplified logic String jwt = parseJwt(request); // Step 1: Extract token if (jwt != null && jwtUtils.validateJwtToken(jwt)) { // Step 2: Is it valid? String username = jwtUtils.getUserName(jwt); // Step 3: Who is this? UserDetails user = userDetailsService.load(username); // Step 4: Load from DB // Step 5: Tell Spring "this user is authenticated" UsernamePasswordAuthenticationToken auth = new UsernamePasswordAuthenticationToken(user, null, user.getAuthorities()); SecurityContextHolder.getContext().setAuthentication(auth); // ✅ Authenticated! } filterChain.doFilter(request, response); // Always pass request forward

💡 Key Insight — SecurityContext
When you call SecurityContextHolder.getContext().setAuthentication(auth), you're telling Spring: "This request belongs to this authenticated user." Every controller in the request chain can now access this user's info and roles.
// Complete Journey
Step-by-Step Request Flow
1
User sends a request
Client makes an HTTP request to a protected endpoint (e.g. GET /api/products) with a JWT in the Authorization header.
Authorization: Bearer eyJhbGci...
2
AuthTokenFilter intercepts it
Since it extends OncePerRequestFilter, it runs before any controller sees the request. It calls parseJwt() to extract the token from the header.
AuthTokenFilter.doFilterInternal()
3
JwtUtils validates the token
validateJwtToken() checks: Is the signature valid? Is it expired? Is it malformed? Returns true or false.
✓ Valid signature ✓ Not expired ✓ Well-formed
4
Username extracted → User loaded from DB
getUserNameFromJwtToken() decodes the token payload to get the username. Then UserDetailsService.loadUserByUsername() fetches full user details (roles, etc.) from your database.
UserDetailsService → Database
5a
No valid token? → AuthEntryPointJwt fires
If no token was found or validation failed, Spring Security detects an unauthenticated request and calls AuthEntryPointJwt.commence(), which returns a 401 JSON error.
HTTP 401 Unauthorized
5b
Valid token → SecurityContext updated → Request continues ✅
The authenticated user object is set in SecurityContextHolder. The filter calls filterChain.doFilter() to pass the request to your controller, which can now access the authenticated user.
Request reaches @RestController ✅
